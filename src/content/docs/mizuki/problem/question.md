本文档帮助你学会如何有效提问，以便更快地获得帮助。

## 为什么需要学习提问？

好的问题能够：
- 更快地获得准确的答案
- 减少沟通成本
- 帮助他人更好地理解你的问题
- 提高解决问题的效率

---

## 提问前的准备

### 1. 自己先尝试解决

在提问之前，请先尝试以下步骤：

- **阅读文档**：查看相关文档和 FAQ
- **搜索问题**：使用搜索引擎搜索错误信息
- **检查日志**：查看错误日志和控制台输出
- **复现问题**：确保问题可以稳定复现

### 2. 收集信息

准备好以下信息：

- **环境信息**：操作系统、Node.js 版本、pnpm 版本
- **错误信息**：完整的错误信息和堆栈跟踪
- **复现步骤**：详细的复现步骤
- **已尝试的解决方案**：你已经尝试过哪些方法

### 3. 最小化问题

尝试创建一个最小的复现示例：

```bash
# 创建最小复现项目
mkdir minimal-repro
cd minimal-repro
pnpm init
# 添加必要的依赖和代码
```

## 如何提问

### 使用正确的格式

#### 标题

标题应该简洁明了，概括问题的核心：

```
❌ 不好的标题
- "求助"
- "出错了"
- "为什么不行"

✅ 好的标题
- "部署到 Vercel 时出现 404 错误"
- "Bangumi API 返回 401 未授权"
- "图片在移动端显示不正确"
```

#### 问题描述

使用以下结构描述问题：

```markdown
## 环境信息

- 操作系统：Windows 11
- Node.js：v22.0.0
- pnpm：v9.0.0
- Mizuki 版本：最新

## 问题描述

简要描述你遇到的问题。

## 复现步骤

1. 执行 `pnpm dev`
2. 访问 http://localhost:4321
3. 点击 xxx 按钮
4. 出现错误

## 期望行为

描述你期望的正确行为。

## 实际行为

描述实际发生的行为。

## 错误信息

```
粘贴完整的错误信息
```

## 已尝试的解决方案

- 尝试了 xxx，没有效果
- 查看了 xxx 文档，没有找到相关说明
```

## 注意事项

1. **礼貌待人**：尊重帮助你的人
2. **耐心等待**：不要重复提问
3. **及时反馈**：问题解决后及时反馈
4. **分享经验**：如果你解决了问题，分享你的解决方案
